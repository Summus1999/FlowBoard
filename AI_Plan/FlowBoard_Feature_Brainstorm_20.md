# FlowBoard 未来功能蓝图 - 20 大功能点头脑风暴

> 角色：产品经理视角
> 产出日期：2026-03-01
> 产品版本基线：v2.0.1
> 目标用户：5 年内社招开发者 + 在校学生
> 核心定位：面向开发者的一站式个人工作台

---

## 0. 现有功能概览（基线）

FlowBoard 当前已具备以下 12 个核心模块：

1. 多平台账户密码管理
2. 界面 UI 风格切换（四套主题 + 毛玻璃）
3. 实时资讯整合（微博/知乎/抖音/头条）
4. Markdown 笔记编辑器
5. 日程管理日历
6. LeetCode 刷题集成（Monaco Editor + 智能提示）
7. GitHub 项目追踪
8. 应用中心（本地应用快速启动 + 图标 DIY + 拖拽排序）
9. 面试追踪（录音 + 笔记 + Checklist）
10. 个人提升（学习计划导入 + 模板 + 定时提醒）
11. 实时天气
12. 系统设置（开机启动/最小化行为）

AI 后端（ai_service）已完成架构设计，基于 FastAPI + LangGraph + LangChain + PostgreSQL + Redis 技术栈。

以下 20 个功能点是在此基线之上的扩展方向。

## 功能 1：AI 对话助手

### 1.1 功能定位

在 FlowBoard 中嵌入一个全局可唤起的 AI 对话面板，作为所有 AI 能力的统一入口。用户可以用自然语言完成知识问答、任务下达、计划生成等操作，不再需要在各模块间手动切换。

### 1.2 交互逻辑

1. 唤起方式
   - 侧边栏固定入口：「AI 助手」栏目
   - 全局快捷键：`Ctrl/Cmd + J` 呼出悬浮对话面板
   - 任意页面右下角的悬浮气泡按钮

2. 对话面板布局
   - 左侧：历史会话列表（按时间倒序，支持搜索和置顶）
   - 右侧：当前对话区域
     - 顶部：会话标题 + 模型标识 + 新建会话按钮
     - 中部：消息流（支持 Markdown 渲染、代码高亮、引用卡片）
     - 底部：输入框 + 文件上传按钮 + 发送按钮

3. 消息流交互
   - AI 回复采用 SSE 流式输出，逐字显示，带打字机动画
   - 每条 AI 回复底部展示：复制按钮、重新生成按钮、点赞/点踩按钮
   - 引用来源以内联卡片形式展示，点击可跳转到原文档对应位置
   - 低置信度回复（< 90%）顶部显示黄色风险提示横幅

4. 多轮对话
   - 支持上下文连续对话，自动维护会话记忆
   - 用户可手动清除上下文（「新话题」按钮）
   - 长会话自动摘要，避免 token 溢出

5. 意图分流
   - AI 自动识别用户意图并路由到对应 Agent：
     - 知识问答 -> RAG QA Agent
     - 制定计划 -> Planner Agent
     - 任务拆解 -> Decomposer Agent
     - 进度复盘 -> Review Agent
   - 路由结果在消息流中以小标签形式透出（如「正在使用知识问答模式」）

### 1.3 关键规则

- 对话数据本地持久化，支持导出为 Markdown
- 单次对话 token 上限提示，超限时引导用户开启新会话
- 模型切换入口在会话顶部，切换后当前会话保持连续性

## 功能 2：智能学习规划引擎

### 2.1 功能定位

基于 AI Planner Agent，将当前个人提升模块从「被动导入计划」升级为「AI 主动生成计划」。用户描述学习目标后，AI 生成结构化的学习计划，经用户确认后进入执行态。

### 2.2 交互逻辑

1. 计划生成流程
   - 用户在「个人提升」页面点击「AI 生成计划」按钮
   - 弹出目标输入对话框：
     - 学习目标（必填，如「三个月内掌握 Go 语言后端开发」）
     - 当前水平（选填，下拉选择：零基础/有基础/进阶）
     - 每周可投入时间（选填，滑块 5~40 小时）
     - 截止日期（选填，日期选择器）
   - 点击「生成计划」后进入等待态，展示生成进度条

2. 计划提案确认
   - AI 返回完整的计划提案，以 Markdown 卡片形式展示
   - 提案内容包含：阶段划分、每阶段目标、推荐资源、里程碑节点
   - 底部操作按钮：
     - 「确认执行」：计划进入执行态，自动写入日程和待办
     - 「修改建议」：打开对话框输入调整意见，AI 重新生成
     - 「放弃」：丢弃当前提案

3. 计划版本管理
   - 每次修改生成新版本，版本号递增
   - 支持版本对比视图：左右双栏 diff 展示变更内容
   - 支持一键回滚到任意历史版本
   - 版本列表以时间线形式展示

4. 计划执行态
   - 计划条目转化为可勾选的任务列表
   - 每个任务支持标记状态：待开始 / 进行中 / 已完成 / 已跳过
   - 整体进度条实时更新
   - 到达里程碑节点时弹出成就提示

### 2.3 关键规则

- 计划创建和覆盖更新必须经过用户确认（提案-确认二步制）
- 计划版本保留最近 5 个版本
- 计划确认后自动同步到日程管理和待办事项模块

## 功能 3：任务拆解与智能看板

### 3.1 功能定位

将学习计划中的大目标自动拆解为可执行的小任务，并以看板视图管理任务流转状态。弥补当前仅有待办列表缺乏结构化管理的短板。

### 3.2 交互逻辑

1. 任务拆解入口
   - 在已确认的学习计划中，点击任意阶段的「AI 拆解」按钮
   - AI Decomposer Agent 自动将该阶段拆解为 5~15 个具体任务
   - 每个任务包含：标题、预估耗时、难度标签、前置依赖关系

2. 看板视图
   - 四列布局：待办 | 进行中 | 待复盘 | 已完成
   - 任务卡片内容：标题、预估时间、难度色标、所属计划标签
   - 交互操作：
     - 拖拽卡片在列之间移动，实时更新状态
     - 点击卡片展开详情侧边栏
     - 右键菜单：编辑 / 删除 / 设置截止日期 / 添加子任务

3. 批量操作
   - 支持多选（Shift + 点击 或 框选）
   - 批量修改状态、批量设置截止日期、批量删除
   - 批量删除和批量修改操作触发二次确认弹窗：
     - 展示操作影响范围（N 个任务将被修改/删除）
     - 用户必须点击「确认执行」才会生效

4. 过滤与排序
   - 按计划、难度、截止日期、状态多维度过滤
   - 排序选项：截止日期优先 / 难度优先 / 创建时间

5. 任务依赖可视化
   - 支持切换到甘特图视图，展示任务时间线和依赖关系
   - 依赖关系用箭头连线表示
   - 关键路径高亮显示

### 3.3 关键规则

- 删除操作不可逆，删除前必须二次确认
- 批量修改必须二次确认
- 任务状态变更自动记录操作日志
- 任务完成后自动更新关联学习计划的进度

## 功能 4：知识库与文档管理

### 4.1 功能定位

为 RAG 系统提供前端管理界面，让用户可以方便地管理个人知识库。支持本地文档导入、索引状态监控、文档搜索与浏览。

### 4.2 交互逻辑

1. 知识库入口
   - 侧边栏新增「知识库」栏目
   - 进入后展示知识库仪表盘：文档总数、索引状态、最近更新

2. 文档导入
   - 支持三种导入方式：
     - 拖拽文件到上传区域
     - 点击「选择文件」按钮，支持多选
     - 指定本地目录路径，自动扫描
   - 支持格式：pdf / docx / txt / md / 音频文件（mp3/wav/webm）
   - 导入后显示处理进度：解析中 -> 清洗中 -> 分块中 -> 索引中 -> 完成
   - 音频文件额外显示 ASR 转写进度

3. 文档列表
   - 表格视图：文件名、类型、大小、分块数、索引状态、导入时间
   - 支持搜索（文件名 + 内容全文搜索）
   - 支持按类型、状态、时间过滤
   - 点击文档名展开预览侧边栏，显示文档内容摘要

4. 索引管理
   - 索引版本列表：版本号、创建时间、文档数、分块数、当前激活标识
   - 操作：
     - 「触发增量索引」：对变更文件重新索引
     - 「触发全量重建」：全量重新索引（带二次确认）
     - 「切换激活版本」：点击指定版本，设为当前检索使用的版本
     - 「回滚」：回滚到指定历史版本

5. 文档质量看板
   - 展示低质量文档列表（质量分低于阈值的分块）
   - 支持人工审核：确认保留 / 标记排除
   - 被排除的分块不参与检索

### 4.3 关键规则

- 全量重建操作需二次确认
- 保留最近 5 个索引版本
- 文件变更采用增量同步，基于文件快照比对（path + size + mtime + sha256）

## 功能 5：学习进度复盘系统

### 5.1 功能定位

定期对用户的学习进度进行结构化复盘，由 Review Agent 生成复盘报告，帮助用户发现学习瓶颈、调整节奏。

### 5.2 交互逻辑

1. 复盘触发方式
   - 自动触发：每周日晚 20:00 推送复盘提醒通知
   - 手动触发：在个人提升页面点击「生成复盘报告」按钮
   - AI 对话中输入「帮我做一下本周复盘」

2. 复盘报告内容
   - 报告以 Markdown 卡片渲染，包含以下模块：
     - 本期概览：计划完成率、任务完成数、学习总时长
     - 目标达成分析：每个里程碑的完成情况，用进度条可视化
     - 效率分析：每日学习时长柱状图、完成任务趋势折线图
     - 瓶颈识别：AI 分析未完成任务的原因模式（如「时间分配不均匀」「某模块难度过高」）
     - 调整建议：基于分析给出下期学习节奏调整方案
     - 鼓励语：根据完成情况生成个性化激励文案

3. 复盘历史
   - 时间线视图展示所有历史复盘报告
   - 支持按时间范围筛选
   - 支持导出为 PDF

4. 趋势追踪
   - 多期复盘数据自动聚合
   - 展示长期趋势图：月度完成率、累计学习时长、技能雷达图变化

### 5.3 关键规则

- 复盘报告基于真实学习数据生成，不允许编造数据
- 报告中的建议以提案形式展示，用户可选择采纳或忽略
- 采纳建议后自动调整当前学习计划（需确认）

## 功能 6：面试 AI 复盘

### 6.1 功能定位

扩展当前面试追踪模块，对面试录音进行 AI 智能分析，生成面试复盘报告。包含表现评估、知识点覆盖度分析、改进建议。

### 6.2 交互逻辑

1. 录音分析入口
   - 在面试追踪页面，每条录音记录右侧新增「AI 分析」按钮
   - 点击后开始分析流程：
     - 第 1 步：ASR 转写（展示进度百分比和实时转写文本）
     - 第 2 步：内容分析（提取关键问答对）
     - 第 3 步：报告生成

2. 转写文本编辑
   - 转写完成后，用户可修正转写文本（编辑模式）
   - 修正后的文本用于后续分析，提升准确度
   - 标注说话人角色：面试官 / 候选人（支持自动识别 + 手动修正）

3. 复盘报告
   - 问答对提取：逐条列出面试问题与回答
   - 表现评分：
     - 技术深度（1~5 星）
     - 表达清晰度（1~5 星）
     - 逻辑完整性（1~5 星）
     - 综合评分
   - 知识点覆盖：提取涉及的技术知识点，标记掌握/待加强
   - 改进建议：针对薄弱环节给出具体学习建议
   - 推荐题目：关联 LeetCode 推荐相关算法题

4. 报告操作
   - 导出为 PDF / Markdown
   - 一键将薄弱知识点添加到学习计划
   - 将推荐题目添加到 LeetCode 收藏列表

### 6.3 关键规则

- 录音分析仅在本地处理（ASR 调用云端模型但不存储原始音频）
- 转写文本支持版本管理
- 分析结果与面试记录关联存储

## 功能 7：番茄钟与专注模式

### 7.1 功能定位

提供沉浸式学习/工作环境。通过番茄钟计时法帮助用户保持专注，自动记录专注时长并纳入学习统计。

### 7.2 交互逻辑

1. 番茄钟入口
   - 顶部工具栏常驻小时钟图标，点击展开番茄钟面板
   - 也可在任意任务卡片上点击「开始专注」按钮启动

2. 番茄钟面板
   - 圆形进度环显示倒计时（默认 25 分钟工作 / 5 分钟休息）
   - 面板内容：
     - 当前关联任务名称（可选）
     - 倒计时数字
     - 操作按钮：开始 / 暂停 / 放弃 / 跳过休息
     - 今日番茄数统计（小番茄图标）

3. 专注模式
   - 启动番茄钟后可切换到「专注模式」
   - 专注模式效果：
     - 全屏遮罩，仅保留当前工作区域
     - 屏蔽资讯中心等可能分心的模块
     - 底部显示简约的倒计时条
     - 专注结束时播放提示音 + 桌面通知

4. 参数设置
   - 自定义工作时长（15~60 分钟）
   - 自定义短休息时长（3~15 分钟）
   - 自定义长休息时长（15~30 分钟）
   - 每 N 个番茄后进入长休息（默认 4 个）
   - 提示音选择（铃声/鸟鸣/静音）

5. 数据记录
   - 每个完成的番茄自动记录：开始时间、结束时间、关联任务
   - 数据自动纳入学习统计（学习时长 = 完成的番茄总时长）
   - 每日/每周番茄数热力图展示

### 7.3 关键规则

- 放弃的番茄不计入统计
- 关联任务的番茄时长自动累加到任务耗时
- 专注模式下仍允许紧急通知穿透

## 功能 8：全局搜索与命令面板

### 8.1 功能定位

打造类 VS Code 的命令面板体验，用户可以通过一个统一入口快速搜索所有内容、执行操作、跳转页面，大幅提升操作效率。

### 8.2 交互逻辑

1. 唤起方式
   - 快捷键：`Ctrl/Cmd + K`（已有，升级交互）
   - 点击顶部搜索栏

2. 搜索面板
   - 居中弹出的搜索框（类似 Spotlight / VS Code 命令面板）
   - 输入时实时显示搜索结果，按类别分组：
     - 页面：匹配侧边栏所有页面名称，回车直接跳转
     - 笔记：匹配笔记标题和内容
     - 任务：匹配待办事项和任务看板
     - 日程：匹配日程标题
     - 知识库：全文检索知识库文档
     - 命令：匹配可执行的操作命令

3. 命令模式
   - 输入 `>` 前缀进入命令模式
   - 可用命令示例：
     - `> 新建笔记` -> 创建笔记
     - `> 新建待办` -> 添加待办
     - `> 切换主题` -> 展示主题选项
     - `> 开始专注` -> 启动番茄钟
     - `> 打开设置` -> 跳转设置页面
     - `> AI 助手` -> 打开 AI 对话面板

4. 最近访问
   - 搜索框为空时，展示最近访问记录（页面、笔记、任务）
   - 支持键盘方向键导航 + 回车确认

5. 搜索结果交互
   - 每条结果显示：图标 + 标题 + 所属模块标签 + 内容摘要
   - 键盘上下键切换选中项，回车执行
   - ESC 关闭面板

### 8.3 关键规则

- 搜索实时响应，输入防抖 200ms
- 搜索结果按相关度排序，最多展示每类 5 条
- 搜索历史本地存储，支持清除

## 功能 9：通知中心与智能提醒

### 9.1 功能定位

统一管理 FlowBoard 中所有模块的通知和提醒，避免分散在各模块的提醒被遗漏。支持智能提醒——AI 根据用户行为模式主动推送提醒。

### 9.2 交互逻辑

1. 通知入口
   - 顶部工具栏铃铛图标，未读数红点标识
   - 点击展开通知侧边面板

2. 通知面板
   - 分为三个 Tab：全部 / 未读 / 已归档
   - 每条通知包含：图标、标题、摘要、时间、来源模块标签
   - 操作：标为已读 / 归档 / 点击跳转到关联页面
   - 底部「全部标为已读」按钮

3. 通知来源
   - 日程提醒：日程到期前 N 分钟推送
   - 学习提醒：定时学习提醒
   - 任务提醒：任务截止日期临近提醒
   - AI 复盘提醒：每周复盘提醒
   - 番茄钟提醒：专注结束 / 休息结束
   - 系统通知：索引完成、导入完成等

4. 智能提醒（AI 驱动）
   - 学习节奏异常：连续 3 天未学习时推送激励提醒

## 功能 11：数据同步与云备份

### 11.1 功能定位

### 11.2 交互逻辑

1. 同步设置入口
   - 系统设置页面新增「数据同步」区域

   - 备份目标：
     - 本地备份：导出到用户指定目录（JSON 压缩包）

     - WebDAV：支持坚果云、自建 NAS 等 WebDAV 服务
   - 备份频率：手动 / 每日自动 / 每周自动
   - 备份范围：全量 / 选择模块（笔记、日程、任务、知识库等）

- 恢复操作必须二次确认
- 本地备份保留最近 10 个版本
- 云备份默认加密传输

## 功能 15：自动化工作流引擎

### 15.1 功能定位

允许用户创建「如果...那么...」的自动化规则，将重复性操作串联起来。类似 IFTTT / Zapier 的本地版。

### 15.2 交互逻辑

1. 工作流入口
   - 系统设置中新增「自动化」区域
   - 或在命令面板中搜索「工作流」

2. 工作流创建
   - 可视化规则编辑器，上下两部分：
     - 触发条件（WHEN）：
       - 时间触发：每天 / 每周 / 特定时间
       - 事件触发：任务完成时 / 日程到期时 / 习惯打卡时 / 文件变更时
       - 条件触发：连续 N 天未学习 / 任务完成率低于 X%
     - 执行动作（THEN）：
       - 创建待办
       - 发送通知
       - 触发备份
       - 调用 AI 生成复盘
       - 更新学习计划
       - 打开指定页面

   - 「每日晨间计划」：每天 8:00 推送今日任务清单
   - 「周日复盘」：每周日 20:00 自动触发 AI 复盘

   - 「文档自动索引」：知识库目录文件变更时自动触发增量索引

4. 工作流管理
   - 列表展示所有工作流
   - 每个工作流可单独启用/禁用
   - 执行日志：展示每次触发的时间、执行结果、错误信息

### 15.3 关键规则

- 工作流执行失败不影响应用主流程
- 工作流执行有频率限制（防止循环触发）
- 涉及数据修改的动作需在工作流创建时确认授权

      - 时间戳转换
      - JSON 格式化

   c. 颜色工具
      - 取色器
      - 颜色格式转换（HEX / RGB / HSL）
      - 最近使用颜色历史

   d. 正则表达式测试器
      - 左侧输入正则表达式
      - 右侧输入测试文本
      - 实时高亮匹配结果
      - 常用正则模板库

   e. 快捷翻译
      - 中英互译
      - 选中文本一键翻译
      - 技术术语优化翻译

   f. Markdown 转换
      - Markdown 转 HTML
      - HTML 转 Markdown
      - 实时预览

3. 工具唤起
   - 命令面板搜索工具名称直接打开
   - 每个工具支持弹出为独立小窗口
   - 支持自定义快捷键绑定

### 19.3 关键规则

- 所有工具离线可用
- 工具窗口支持「置顶」模式
- 剪贴板历史敏感内容（如密码）自动过滤

## 功能 20：社区与学习资源推荐

### 20.1 功能定位

打通「个人学习」和「社区互动」，为用户推荐高质量学习资源，并支持轻量级的社区互动。从个人工具向知识社区进化的第一步。

### 20.2 交互逻辑

1. 社区入口
   - 侧边栏新增「社区」栏目

2. 学习资源推荐
   - 基于用户学习计划和技能方向，AI 推荐相关学习资源：
     - 免费课程链接
     - 优质技术博客
     - 开源项目推荐
     - 技术书籍推荐
   - 资源卡片展示：标题、来源、推荐理由、评分
   - 操作：添加到书签 / 添加到学习计划 / 忽略

3. 学习路径分享
   - 用户可将自己的学习计划公开为模板
   - 其他用户可浏览和导入公开的学习路径
   - 支持点赞和评论

4. 每日推荐
   - 每日推送一道 LeetCode 每日一题
   - 每日推送一篇技术精选文章
   - 每日推送一个「今日知识点」（基于学习计划的知识点提炼）

5. 学习圈子
   - 按学习方向分圈：前端、后端、算法、系统设计、AI
   - 圈子内支持：发帖、回复、点赞
   - 管理员审核机制保证内容质量

6. 匿名经验分享
   - 面试经验分享（脱敏后匿名发布）
   - 学习方法分享
   - 踩坑记录分享

### 20.3 关键规则

- 社区功能为可选模块，用户可选择不开启
- 分享的内容默认匿名
- 社区数据存储在云端，需联网使用
- 内容审核机制防止低质量或违规内容

## 总结

以上 20 个功能点覆盖了 FlowBoard 从「个人效率工具」向「AI 驱动的智能学习工作台」进化的完整路径：

1. 功能 1~4 构建 AI 核心能力层（对话、规划、任务、知识库）
2. 功能 5~7 完善学习闭环（复盘、面试分析、专注管理）
3. 功能 8~9 夯实操作效率基础设施（搜索、通知）
4. 功能 10~12 增强用户留存与习惯养成
5. 功能 13~16 构建平台化与高级交互能力
6. 功能 17~19 扩展工具广度与数据深度
7. 功能 20 探索社区化方向

整体路线图建议以 4~6 个月为一个大版本周期，P0 功能在第一个周期内完成，P1 在第二个周期，P2/P3 视用户反馈和资源情况灵活调整。
