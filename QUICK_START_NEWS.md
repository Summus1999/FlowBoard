# 实时资讯功能 - 快速使用指南

## 🚀 立即使用

### 1. 启动应用

```bash
npm start
```

应用会自动获取所有资讯源的内容（约需 5-15 秒）。

### 2. 手动刷新资讯

在资讯页面点击右上角的 **"🔄 立即刷新"** 按钮。

### 3. 查看订阅源状态

右侧边栏会显示当前启用的资讯源列表。

## ⚙️ 配置说明

### 资讯更新频率

- **自动更新**: 每 4 小时一次
- **手动更新**: 随时点击刷新按钮
- **启动更新**: 应用启动时自动获取

### 显示数量限制

- **总资讯数**: 最多显示 300 条
- **单源数量**: 每个源最多获取 60 条

### 超时设置

- **单个请求超时**: 12 秒
- 超过时限自动跳过，不影响其他源

## 🛠️ 故障排查

### 问题 1: 资讯一直显示"加载中..."

**原因：** 网络延迟或部分源无法访问

**解决方案：**

1. 等待 15-30 秒，让系统完成所有源的获取
2. 如果仍然卡住，点击刷新按钮重试
3. 运行 `npm run test-rss` 检查哪些源有问题

### 问题 2: 资讯数量为 0

**原因：** 所有源都失败或缓存损坏

**解决方案：**

```bash
# 1. 清除缓存
npm run clear-news-cache

# 2. 测试资讯源
npm run test-rss

# 3. 重启应用
npm start
```

### 问题 3: 只有部分资讯（如只有国内源）
**原因：** 国外源网络连接失败

**解决方案：**
1. 检查网络连接
2. 确认防火墙设置
3. 这是正常现象，系统会自动使用可用的源

### 问题 4: 刷新后内容没变化
**原因：** 缓存未更新

**解决方案：**
```bash
npm run clear-news-cache
npm start
```

## 📝 技术细节

### 数据存储位置
- **Windows**: `%APPDATA%\FlowBoard\news-cache-v1.json`
- **macOS**: `~/Library/Application Support/FlowBoard/news-cache-v1.json`
- **Linux**: `~/.config/FlowBoard/news-cache-v1.json`

### 支持的格式
- RSS 2.0
- Atom 1.0

### 去重机制
系统会根据 URL 和标题自动去重，确保不会显示重复内容。

## 💡 最佳实践

1. **定期清理缓存**
   - 建议每周清理一次，确保内容新鲜度
   ```bash
   npm run clear-news-cache
   ```

2. **测试资讯源**
   - 发现内容少或加载慢时，先运行测试
   ```bash
   npm run test-rss
   ```

3. **关注日志输出**
   - 应用启动时查看控制台，了解哪些源成功/失败

4. **合理期待**
   - 不是所有源都能 100% 成功
   - 系统会自动优化，优先显示成功的内容
