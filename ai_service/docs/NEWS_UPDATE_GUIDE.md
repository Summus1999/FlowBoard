## 使用方法

### 启动应用后自动刷新
应用启动时会自动获取所有资讯源的 RSS 内容，每 4 小时自动更新一次。

### 手动刷新
点击资讯页面右上角的"立即刷新"按钮，可手动触发即时刷新。

### 测试资讯源可用性
可以运行以下命令测试所有 RSS 源的连接状态：

```bash
npm run test-rss
```

这将输出每个资讯源的：
- ✓ 成功状态（响应时间、获取条数）
- ✗ 失败状态（错误原因）

## 注意事项

1. **网络环境要求**
   - 部分国外资讯源可能需要稳定的网络连接
   - 如果某些源无法访问，系统会自动跳过并继续获取其他源

2. **性能优化**
   - 增加了超时机制（12 秒），防止单个源卡住整体流程
   - 使用并发获取，提高整体加载速度
   - 去重机制避免重复内容

3. **容错机制**
   - 即使部分源失败，也会继续显示其他成功源的内容
   - 保留历史缓存，确保在网络不佳时仍有内容显示

## 故障排查

### 问题：点击刷新没反应
**解决方案：**
1. 检查网络连接
2. 查看控制台是否有错误信息
3. 尝试重启应用

### 问题：资讯数量为 0
**解决方案：**
1. 运行 `npm run test-rss` 测试资讯源
2. 检查防火墙设置
3. 清除缓存文件（位于用户数据目录的 `news-cache-v1.json`）

### 问题：部分源加载失败
**解决方案：**
这是正常的，因为部分国外源可能受网络环境影响。系统会自动使用其他成功的源来补充内容。
