## 产品概述

FlowBoard 是一款面向开发者和个人用户的桌面工作台应用，采用 Electron + Web 技术栈构建，提供 glassmorphism（毛玻璃）设计风格的现代化界面。产品整合了账户管理、AI 智能助手、学习追踪、日程规划、技术刷题、应用启动等多功能于一体，帮助用户高效管理数字生活。

### 产品亮点

- 🤖 **CrewAI 多智能体系统**：规划师、任务拆解专家、学习复盘师智能协作
- 💬 **多模型 AI 助手**：支持通义千问、Kimi、智谱 GLM、硅基流动，自动故障转移
- 📚 **知识库 RAG**：文档导入、向量检索、智能问答
- 📊 **智能学习规划**：自动生成学习计划、任务拆解、进度复盘
- 🎨 **玻璃态 UI**：现代化设计风格，4种主题可选

### 设计理念

- **一站式管理**：减少在不同应用间切换的成本
- **AI 驱动**：CrewAI 智能体辅助学习和任务管理
- **数据本地化**：优先本地存储，保护隐私
- **开发者友好**：针对程序员场景优化（LeetCode、GitHub 集成）
- **智能辅助**：代码智能提示，提升刷题效率

## 核心功能详解

### 1. 我的主页

**入口**：侧边栏「我的主页」或底部 🏠 房子按钮

**功能组件**：

| 组件 | 功能描述 | 交互方式 |
|------|----------|----------|
| 账户安全概览 | 显示密码管理入口，安全评分圆环 | 点击「查看全部」进入账户密码页面 |
| 待办事项 | 显示全部待办任务（支持滚动浏览） | 点击复选框完成/取消，点击 × 删除，点击「添加待办」新增 |
| 热榜中心 | 显示热门新闻头条，支持科技/财经/社交分类切换 | 点击跳转外部链接 |

**顶部统计栏**：

- 账户密码数量统计
- 待办任务数量统计（未完成的待办数）

**交互逻辑**：

- 主页是应用启动后的默认页面
- 所有模块数据与其他页面实时同步
- 待办事项仅在主页显示，移除独立页面入口

### 2. 账户密码管理

**入口**：侧边栏「账户密码」或主页「查看全部」

**预设账户**（12个国内常用账号）：
微信、支付宝、LeetCode 中国、QQ、微博、淘宝、京东、Bilibili、网易云音乐、百度、GitHub、抖音

**功能特性**：

- **分类管理**：社交/金融/工作/娱乐
- **CRUD 操作**：
  - 添加：点击「添加账户」→ 填写平台/用户名/密码/分类 → 保存
  - 编辑：点击卡片上的编辑图标 → 修改信息 → 保存
  - 删除：点击删除图标 → 确认删除
- **密码强度检测**：添加时自动计算（弱/中/强）
  - 评分规则：长度≥8位+1分，≥12位+1分，包含大小写+1分，包含数字+1分，包含特殊字符+1分
- **快捷操作**：点击复制图标可复制用户名/密码
- **安全评分**：主页显示安全评分圆环（基于密码强度计算）

### 3. 资讯中心

**入口**：侧边栏「资讯中心」

**功能特性**：

| 分类 | 说明 |
|------|------|
| 全部 | 显示所有类别新闻 |
| AI | 追踪 AI 行业最新动态 |
| 科技 | 科技行业新闻 |
| 财经 | 财经新闻 |
| 娱乐 | 娱乐资讯 |
| 社会 | 社会热点 |

**数据来源**：26 个优质 RSS 源（17 国外 + 9 国内）

- 国外：Hacker News、The Verge、TechCrunch、WIRED、BBC、CNN、Bloomberg 等
- 国内：IT 之家、36 氪、虎嗅、钛媒体、财新、凤凰网等

**展示与更新**：

- 最多显示 300 条，每源最多 60 条
- 自动更新（每 4 小时）+ 手动刷新
- 本地缓存，离线可阅读历史内容

**工具链**：

- `npm run test-rss`：测试所有 RSS 源可用性
- `npm run clear-news-cache`：清除资讯缓存

### 4. AI 智能助手

**入口**：侧边栏「AI 助手」

#### 4.1 多会话聊天

**功能特性**：

- **多会话管理**：支持创建、切换、删除、重命名会话，会话列表侧边栏展示
- **流式输出**：实时流式显示模型回复
- **多模型路由**：通义千问、Kimi、智谱 GLM、硅基流动，按配置优先级自动 failover
- **会话持久化**：IndexedDB 存储会话与消息，支持空消息清理脚本

#### 4.2 知识库 RAG

**功能特性**：

- **文档导入**：支持 PDF、Word、Markdown、TXT 格式
- **智能分块**：自动分块和向量化
- **检索问答**：基于文档内容的智能问答
- **来源引用**：回答附带引用来源

**使用方式**：

1. 进入知识库页面上传文档
2. 在 AI 助手中选择「知识库问答」模式
3. 输入问题，AI 基于文档内容回答

#### 4.3 AI 服务配置

**配置入口**：设置页面 → AI 服务配置

**配置项**：

| 配置项 | 说明 |
|--------|------|
| 服务地址 | AI 服务地址（默认 http://localhost:8000） |
| API Token | 远程访问认证（可选） |
| 模型提供商 | 通义千问、Kimi、智谱 GLM、硅基流动 |
| 调用优先级 | 拖拽排序，任一可用即可成功 |
| 月度预算 | 成本控制 |

**安全特性**：

- API Key 使用 Electron safeStorage 加密存储
- 支持连接测试
- 配置热更新，无需重启服务

### 1. 规划师智能体 (Planner Agent)

**角色**：学习计划规划师  
**职责**：根据用户目标制定结构化学习计划

#### 功能特性

- **目标分析**：分析学习目标，识别核心技能需求
- **里程碑规划**：将长期目标拆分为 3-5 个可管理的里程碑
- **任务列表生成**：为每个里程碑生成具体任务
- **风险评估**：识别潜在风险并提供应对策略
- **时间安排**：根据每周可用时间制定学习计划

#### 使用入口

- 个人提升页面 → 智能规划
- AI 助手 → "/plan" 命令

#### 输出示例

```json
{
  "overview": "本计划旨在帮助你从Python基础进阶到后端开发...",
  "milestones": [
    {
      "title": "Python基础强化",
      "duration_days": 14,
      "deliverables": ["完成Python高级特性学习"],
      "success_criteria": "能够熟练使用Python高级特性"
    }
  ],
  "tasks": [...],
  "risk_assessment": [...]
}
```

---

### 2. 任务拆解专家 (Decomposer Agent)

**角色**：任务拆解专家  
**职责**：将复杂任务分解为可执行的子任务

#### 功能特性

- **复杂度评估**：评估任务复杂度（简单/中等/复杂）
- **任务拆解**：将大任务拆分为 30分钟-4小时 的可执行单元
- **依赖分析**：识别任务间的依赖关系和阻塞点
- **关键路径计算**：优化执行顺序
- **并行识别**：识别可并行执行的任务组

#### 使用入口

- 任务看板 → 任务详情 → 智能拆解
- AI 助手 → "/decompose" 命令

#### 输出示例

```json
{
  "complexity_assessment": {
    "level": "medium",
    "needs_decomposition": true
  },
  "subtasks": [
    {
      "title": "设计用户数据模型",
      "estimated_minutes": 60,
      "dependencies": [],
      "checklist": ["定义SQLAlchemy模型", "添加字段验证"]
    }
  ],
  "dependency_analysis": {
    "critical_path": ["task-1", "task-3"],
    "parallel_groups": [["task-1", "task-2"]]
  }
}
```

### 3. 学习复盘师 (Reviewer Agent)

**角色**：学习进度复盘师  
**职责**：分析学习进度并生成复盘报告

#### 功能特性

- **成就识别**：从学习数据中识别值得表扬的成就
- **挑战分析**：分析未完成任务和遇到的困难
- **进度评估**：评估与计划的对比（超前/正常/落后）
- **改进建议**：提供具体可执行的优化措施
- **下阶段目标**：设定合理的后续目标

#### 支持周期

- **日复盘**：每日学习总结
- **周复盘**：每周进度回顾
- **月复盘**：月度学习分析
- **里程碑复盘**：阶段目标完成总结

#### 使用入口

- 个人提升页面 → 学习统计 → 生成复盘
- AI 助手 → "/review" 命令

#### 输出示例

```json
{
  "summary": {

    "key_highlights": ["连续5天保持学习"]

    {
      "type": "连续坚持",
      "title": "连续5天学习",
      "description": "建立了稳定的学习节奏"
    }
  ],
  "challenges": [...],
  "suggestions": [...],
  "next_goals": [...]
}
```

## 交互逻辑汇总

### 导航交互

```javascript
侧边栏点击 → showPage(pageName) 或 onEnter 回调 → 切换 page 显示 → 更新导航激活状态
                        ↓
            触发对应页面初始化函数（如果有）
```

### 侧边栏热插拔

侧边栏由 `sidebar-registry.js` 注册中心驱动，支持：

- **显示/隐藏**：工具分组下点击 👁️/🙈 切换栏目可见性
- **新增栏目**：填写名称、页面标识、图标、onEnter 函数
- **删除**：自定义栏目可删除，内置栏目仅可隐藏
- **持久化**：配置存储于 `flowboard_sidebar_layout_v1`，重启后自动恢复

## 数据存储

### 存储架构

```
┌─────────────────────────────────────────────────────────────┐
│                        FlowBoard                             │
│                                                              │
│   ┌──────────────────┐  ┌──────────────────┐              │
│   │   Renderer       │  │     Main         │              │
│   │   Process        │  │   Process        │              │
│   └────────┬─────────┘  └────────┬─────────┘              │
│            │                     │                         │
│   ┌────────▼─────────┐  ┌────────▼─────────┐              │
│   │ localStorage     │  │   userData       │              │
│   │ IndexedDB        │  │   (Electron)     │              │
│   └──────────────────┘  └────────┬─────────┘              │
│                                  │                         │
│                        ┌─────────▼─────────┐              │
│                        │  AI Service       │              │
│                        │  - SQLite DB      │              │
│                        │  - ChromaDB       │              │
│                        └───────────────────┘              │
│                                                            │
└─────────────────────────────────────────────────────────────┘
```

### localStorage 键值表

| Key | 用途 | 数据格式 |
|-----|------|----------|
| `todos` | 待办事项数据 | JSON Array |
| `flowboard_notes` | 笔记内容 | JSON Array |
| `flowboard_events` | 日程事件 | JSON Array |
| `flowboard_repos` | GitHub 仓库缓存 | JSON Array |
| `flowboard_apps` | 应用中心配置 | JSON Object (v2) |
| `user_profile` | 用户配置 | JSON Object |
| `user_avatar` | 用户头像 | String (Base64) |
| `github_username` | GitHub 用户名 | String |
| `theme` | 当前主题 | String |
| `flowboard_sidebar_layout_v1` | 侧边栏布局 | JSON Object |

### IndexedDB 存储

| 数据库名 | 对象存储 | 用途 |
|----------|----------|------|
| `FlowBoardInterviewDB` | `recordings` | 面试录音 Blob 数据 |
| `FlowBoardDB` | `chatSessions` / `chatMessages` | AI 助手会话与消息 |
| `FlowBoardDB` | `knowledgeDocs` / `knowledgeChunks` | 知识库文档 |
| `FlowBoardDB` | `tasks` / `habits` / `habitRecords` | 任务看板、习惯追踪 |
| `FlowBoardDB` | `bookmarks` / `codeSnippets` | 书签、代码片段 |

### SQLite 数据库（AI 服务）

| 表名 | 用途 |
|------|------|
| `sessions` | AI 会话 |
| `chat_messages` | 聊天消息 |
| `plans` | 学习计划 |
| `tasks` | 任务 |
| `memories` | 长期记忆 |
| `rag_documents` | RAG 文档 |
